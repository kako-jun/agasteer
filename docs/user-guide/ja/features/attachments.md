# ファイル添付

画像・動画・音声・ZIPをリーフに添付できます。

添付ファイルは、ノートのリポジトリとは別の専用プライベートリポジトリ `{owner}/{リポジトリ名}-media` に保存されます（初回添付時に自動作成）。

---

## 添付の方法

3つの方法があります。いずれもリーフ編集中に使えます。

### 貼り付け

スクリーンショットなどの画像をコピーして、エディタに **Ctrl+V**（Macでは Cmd+V）で貼り付け。

- カーソル位置に挿入される
- 通常のテキスト貼り付けはそのまま動く（ファイルを含むときだけ添付になる）

### ドラッグ&ドロップ

ファイルをエディタ領域にドロップ。

- ドロップした位置に挿入される
- 複数ファイルをまとめてドロップできる

### 添付ボタン

フッター中央の「**🖼️ ファイルを添付**」ボタンからファイルを選択（複数可）。

- カーソル位置に挿入される

---

## 挿入される記法

| 種類                | 記法              |
| ------------------- | ----------------- |
| 画像                | `![名前](rawURL)` |
| 動画・音声・ZIPなど | `[名前](rawURL)`  |

URLは添付した瞬間に確定し、記法の挿入もアップロードの完了を待たずその場で行われます。挿入はアップロード完了後ではなくローカル処理（最適化とキュー登録の数ミリ秒）の直後に走るため、添付した直後に別のリーフへ切り替えたり、プレビュー（👁️）に切り替えたりしても、挿入した記法が失われる窓（すき間）は実質なくなります。アップロードは裏側で続き、失敗してもキューに残って自動で再試行されます。

---

## 対応形式とサイズ上限

- **画像**: png / jpg / jpeg / gif / webp / svg
- **動画**: mp4 / webm
- **音声**: mp3 / m4a / ogg / wav
- **その他**: zip
- **サイズ上限**: 1ファイル100MBまで

形式外・100MB超のファイルは拒否され、トーストで通知されます。

---

## 画像の自動最適化

添付した画像は自動で最適化されます（既定ON）。

- 最大辺2048pxに縮小 + WebP形式に変換
- gif（アニメーション）と svg はそのまま保存
- 設定画面の「**エディタ**」セクションでOFFにできる

> **💡 Tip**: スクリーンショットの直貼りでもファイルサイズが大きく膨らまないので、基本はONのままがおすすめ。

---

## オフラインでも使える

オフラインで添付しても、記法の挿入はその場で完了します。

- ファイルの実体は端末内の保留キューに保存される
- オンラインに復帰すると自動でアップロードされる
- アップロードに失敗した場合もキューに残り、自動で再試行される
- 添付処理中（画像の最適化・保存中）に Push しても、挿入が反映されてから Push されるので、記法だけが保存漏れになることはない

---

## プレビューでの表示

添付ファイルはプレビュー（👁️）で開くと種類に応じて表示されます。

| 種類    | 表示                             |
| ------- | -------------------------------- |
| 画像    | そのまま表示                     |
| 動画    | 再生コントロール付きプレーヤー   |
| 音声    | 再生コントロール付きプレーヤー   |
| ZIPなど | タップでダウンロードできるリンク |

- 取得済みのファイルは即座に、初回は「読み込み中...」の表示を挟んで表示される
- 添付直後（アップロード完了前）やオフラインでも、端末内にあるファイルはそのまま表示される
- 取得できなかった場合（オフラインで未取得・削除済みなど）はファイル名と「**再試行**」ボタンが表示される

## メディアの一覧・削除

添付したメディアは専用画面でまとめて確認・削除できます。

### 開き方

パンくず左端の「**›**」ドロップダウン（ホーム/アーカイブの切り替え）から「**メディア**」を選びます。ホームへ戻るときは、パンくずのホームアイコンか、同じドロップダウンで「ホーム」を選びます。

### 一覧

- 添付済みファイルがサムネイル付きのグリッドで並びます（新しい順）
- 画像は縮小表示、動画・音声・ZIPなどは拡張子ラベルで表示されます
- 各ファイルの名前とサイズが表示されます
- まだ何も添付していない場合は「まだ添付がありません」と表示されます
- 件数の上限は実用上ありません（極端に多い場合のみ「一部のみ表示しています」と表示されます）
- どのノートからも参照されていないファイルには「**未参照**」ラベルが付きます（安心して削除できる目印）。アーカイブをまだ読み込んでいない間は判定が保留され、その旨の案内が表示されます

### 削除

各ファイルの削除ボタン（🗑️）から削除できます。確認ダイアログで「削除しますか？」と尋ねられます。

> **⚠️ 注意**: 削除したメディアがどこかのノートから参照されていると、そのノートの表示が壊れます（画像が出なくなる・リンクが切れる）。「**未参照**」ラベルの付いたファイルは参照されていないため、安心して削除できます。

## 容量について

メディアリポジトリは**履歴を残さない保存方式**です。通常のGitリポジトリと違い、過去のバージョンが蓄積しないため:

- 容量は「いま存在するファイルの合計」だけで済みます
- **削除すると容量も実際に減ります**（履歴に残りません）
- 特別な操作は不要です（自動でこの方式になります）

> **📝 Note**: GitHub上の容量表示への反映は、GitHub側の掃除処理の都合で遅れることがあります。

---

## 注意

- メディアリポジトリはプライベートのため、挿入されたURLをブラウザで直接開いても表示されません（GitHubの認証が必要です）。プレビューでは上記の通り認証付きで取得して表示されます
- **Fine-grained トークン**（セットアップガイドの手順）を使っている場合、メディアリポジトリの自動作成はできません（リポジトリ作成の権限がないため）。`{リポジトリ名}-media` という名前のプライベートリポジトリを自分で作成し、トークンの「Repository access」に**ノートのリポジトリと合わせて追加**してください。Classic トークン（`repo` スコープ）なら初回添付時に自動作成されます
- 「画像としてダウンロード/シェア」には添付画像は含まれますが、動画・音声は映りません
